# Code of Conduct

## The standard

This project is built on adversarial stress-testing. Disagreement is not only welcome — it is the mechanism by which the design improves. The standard for participation is simple:

**Challenge the design, not the person.**

Specific, substantive critique — including critique that finds serious flaws — is the most valuable contribution this project can receive. Personal attack, harassment, and bad-faith engagement are not.

## What this means in practice

**Welcome:**
- Identifying flaws, gaps, or exploits in the protocol's design
- Disagreeing with a patch, clause, or assumption — with mechanism-level reasoning
- Asking hard questions that the current design does not answer well
- Pointing out real-world cases that contradict the design's assumptions

**Not welcome:**
- Personal attacks on contributors or the protocol's author
- Dismissal without substantive engagement ("this will never work")
- Bad-faith participation intended to derail rather than improve
- Harassment of any kind

## Scope

This code of conduct applies to all project spaces — GitHub Issues, Pull Requests, Discussions, and any other venue associated with this project.

## Enforcement

Violations may result in comment removal, temporary restriction, or permanent ban from the project, at the maintainer's discretion. The bar for enforcement is high — this project values difficult conversations. It does not value cruelty.

## Contact

To report a violation, contact the maintainer directly via GitHub.
