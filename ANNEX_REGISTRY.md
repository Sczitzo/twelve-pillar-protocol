# Annex Registry

**Known gap documentation · For navigation and completeness tracking**
**Status: Living document — updated as annexes are created or embedded status changes**

---

## Purpose

The Twelve-Pillar Protocol references approximately 30 named annexes across its documentation. Not all exist as standalone files — many are embedded within `Master_Protocol.md` as numbered sections. This registry documents the location and status of every referenced annex, so reviewers know where to find each one and the founding coalition knows what remains to be extracted or created.

This is a navigation and accountability document. It does not create new obligations; it surfaces existing ones.

---

## Annex Status Key

| Status | Meaning |
| :--- | :--- |
| **Standalone file** | Exists as its own `.md` file in this repository |
| **Embedded** | Exists as a named section within `Master_Protocol.md`; operative but not separately accessible |
| **Pending — pre-launch gate** | Referenced but not yet written; a blocking gate for one or more patches |
| **Pending — recommended** | Referenced but not yet written; not a hard blocking gate but should be created before scale-up |
| **Retired** | Referenced in older versions; superseded or absorbed |

## Embedded Annexes Note

Annexes AB through AH are embedded within `Master_Protocol.md` as numbered sections. They are readable and operative but not separately navigable as standalone files. Extracting them into standalone files is recommended before any public launch for the following reasons:

1. **External reviewers** cannot easily cite or link to specific annex content
2. **Amendment tracking** is harder when operative language is embedded in a large document
3. **Version control** granularity is lost — a change to Annex AC1 requires committing a change to the entire Master_Protocol.md

Extraction is not a blocking gate for any patch but is recommended as part of launch readiness. Priority order for extraction (by how frequently they are cross-referenced): AG (P-013), AH/AH2 (P-014), AC2 (P-006/PCRP), AE (P-012 interface hardening).
